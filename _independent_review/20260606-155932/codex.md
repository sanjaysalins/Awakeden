# Independent review — codex (OK, 191s)

**Findings**
- `"Est. ~$1940 to take everything to a cut"` is not defensible. In the generator this is just fixed `long=$30`, `short=$20` math, while repo docs price images/animation per scene and the Isaiah 53 long used 21 scenes. It ignores retries, captions, thumbnails, upload kits, and failure contingency.

- `"Long TIERING: Tier-1 series-anchor long (1/series) backs the tail"` is asserted, not proven. A generic anchor may not substantiate distinct shorts like Bread, Light, Door, Shepherd, Vine, each with different OT/theological claims. Require a per-short claim ledger with long-form timestamp/source coverage.

- `"Write the long FIRST"` conflicts with `"Shorts awaiting their long ... 73"` and `"existing shorts get their long retrofitted"`. Retrofit can become post-hoc justification. The plan needs an audit step: keep, recut, retire, or rewrite unsupported existing shorts.

- `"Post Isaiah 53 (it is captioned / ready)"` being priority #5 is backwards. Since `"COMPLETE (uploaded) 0"`, the distribution loop is unproven. Publish the ready long and learn before funding more long-form/batch work.

- `"Pipeline: planned -> narration -> mp3 -> stills -> clips -> cut(POC) -> captioned -> uploaded(COMPLETE)"` is only a state list. It omits done definitions: script lock, doctrinal review, audio approval, image gate, clip gate, caption safe-area check, metadata, thumbnail, upload verification, and post-watch QA.

- `"Built from data/series.json"` is only partly true. The 10/76 catalog count is real, but statuses, flagships, off-catalog work, and cost are hardcoded overlays. That makes `"Started 16"` and `"Cut+ 9"` fragile unless there is a machine-readable production register.

- `"Finish Psalm 22 long (scene plan -> stills -> clips -> assemble)"` understates buildability. Local docs say long-form is still hand-driven with no single runner. This needs concrete commands, gates, spend estimate, rollback/backups, and known cross/NSFW/animation failure handling.

- `"Reuse ... realistic reuse ~15-25%"` and `"passion cluster = max reuse"` are unmeasured. There is no scene-level reuse audit, render-once list, asset taxonomy, or acceptance rule for “thread-NEUTRAL” plates before spend.

- Priority step `"Retrofit longs behind the already-cut shorts ... Prodigal"` contradicts `"The Prodigal Son ... NOT a greenlit series.json episode"`. Either greenlight parables or stop spending long-form support on off-catalog work.

- Cross-series duplication is unresolved: `"The Passover Lamb"`, `"Lamb of God"`, `"The Lamb of God"`; `"It is finished"` in two series; John 21:17 in Encounters/QJA; Luke 23:43 in Words/Encounters. Cross-filing is not a dedup/stagger strategy.

- Several topics need episode-specific doctrinal/textual guardrails. `"The woman caught in adultery <John 8:11>"` lacks textual-variant handling. `"The thief on the cross"` is not really “one-on-one.” `"Last Week, Hour by Hour"` promises chronology while warning Gospel timing diverges, but gives no harmonization policy.

- `"Resurrection on Trial"` promises strongest objections, but the plan gives only verse refs. Claims like `"Paul's early creed"` need a bibliography/source ledger and skeptic-strength check, not just a catalog slot.
