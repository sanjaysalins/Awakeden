## Context — why this exists

The Psalm 22 short cluster shipped 8 narrations each individually sound but, **as a set**, templated: **8/8 end "Come to Him"**, plus repeated hooks ("a thousand years…"), bridges ("bring you home"), and tags ("That's Jesus"). This is the *cheesy/tired* failure the project most needs to avoid — and it is the **Word of God**, so sloppiness is unacceptable.

Neither the in-engine red-team nor the external 5-CLI panel caught it. The cause is **structural**:

1. **Every check is per-single-artifact.** Each short was reviewed in isolation; no reviewer ever saw all 8 together. `independent_review.py` takes one `artifact`; `LENS_NARRATION` has no cross-artifact concern.
2. **The engine's gates never ran on this content.** G1–G8, the `_apply_kjv_gate` KJV override, the CTA/landing tests, and the tournament judge all live inside `engine.generate → engine.review`. The 8 shorts and *all* long-form are **hand-authored**, so they bypassed every gate — and **there is no lock step that forces hand-authored content through any check** (`_finalize.py` only renders audio; `handoff.py` only covers engine-generated folders).
3. **The KJV cache can be corrupt and is trusted blindly.** `data/kjv_cache.json` holds `Psalm 22:7` as `"…they shake the head saying,"` — the 1769 KJV has the comma: `"…they shake the head, saying,"`. `kjv_check` compares quotes **against that same cache**, and **never verifies NT cross-refs** (low-overlap spans fall through to "ignored"). (Acts 8:35 is **clean** in the cache — that earlier incident was an authoring elision in the Isaiah text, not cache corruption.)
4. **`tired-landing` is intra-artifact only.** No defect class for cross-artifact/catalogue repetition; the banned-closer list is static (banned "Will you trust Him?"; "Come to Him" became the next cliché).

**The two true root causes, in one line each:**
- **No one is ever forced to look at the set, or the catalogue, before lock** → repetition ships.
- **Scripture is trusted from a corrupt cache with no independent witness** → silent verbatim errors.

Everything below targets exactly those two, at the scale of 10 series / ~76 catalog shorts + ~12–21 long-forms.

## The keystone: a fail-closed LOCK CHOKEPOINT (`cli_lock.py`)

The panel's dominant finding: a standalone CLI nobody is *required* to run reproduces the org failure that shipped this. So the center of the plan is **one non-bypassable lock step** that every narration — engine-generated, hand-authored, short, or long — must pass.

- `cli_lock.py "<narration-folder>"`:
  1. runs the **deterministic verify tier** (KJV ref-mapped span check, Rule-8 [short-only], anchor-verse, banned/lecture phrases) via the single `verify_narration.py` core;
  2. runs the **cluster check** against the series + catalogue (the set-level gate);
  3. on 0 FAIL / 0 unverified, **appends the artifact's phrases to the ledger** and writes a `<folder>/.locked` token (with the commit-ish of the narration text);
  4. exits **non-zero** on any FAIL/unverified — nothing downstream proceeds.
- **Downstream refuses without the token:** `per_turn_synth` / `_finalize.py` / the audio + caption render steps check for a current `.locked` matching the narration hash and **abort** if missing or stale. (This is the enforcement v1/v2 lacked.)
- **Proof obligation:** the 8 Psalm 22 shorts + 2 long-forms must be shown to flow through `cli_lock.py` before any further code is built on top.

This chokepoint is what makes the standing gate real; A2/B/C below are the checks it runs.

## Phase A — PROVE THE CATCH FIRST (the actual miss; ~$0)

### A1 — Shared spoken + ref parser (`pipeline/narration_parse.py`), fail-closed, replaces the old extractor

**Problem (verified, all reviewers).** `veed_io/_extract_spoken.py` keeps text only inside `## MOVEMENT` (CLI-only) → returns **empty** on shorts (`**[narrator]**` blocks) → checks pass on nothing.

**Design.** New importable module:
- `spoken_blocks(md)` → ordered `{speaker, ref|None, text}`, parsing `**[speaker]**` / `**[speaker — KJV, <ref>]**` short blocks AND `## MOVEMENT` long-form sections; strips status front-matter, `## DEPTH & SOURCING`, `## VOICE PLAN`.
- `quoted_spans_with_refs(md)` → each double-quoted span mapped to its block's tagged ref (the quote→ref map B4 needs). **Handles range refs** (`Psalm 22:6-7`, `Isaiah 53:10-11`) and multiple spans per block.
- `sentences(text)` → sentence/clause segmentation. **Normalizes** smart quotes/dashes/whitespace.
- **Fail-closed:** zero spoken blocks → raise; callers ABORT.
- **Retire `veed_io/_extract_spoken.py`**: delete it and rewire all `veed_io` callers (caption/aligner/timing) to import `narration_parse`, so there is ONE spoken-text source (no caption/verification drift).
- **Acceptance:** unit test asserts each of the 8 shorts + 2 long-forms parses to non-empty spoken text **and every KJV quote in them is successfully ref-mapped** (unmapped span = test failure, not silent human-check).

### A2 — Cluster / series check (a mode of `verify_narration.py`)

**Deterministic core ($0):**
- Per artifact extract via A1: **CTA** (last spoken sentence/clause), **hook** (first 1–2 spoken sentences), **opener family** ("thousand years" and similar stems anywhere in the opening block — *not* just first 6 tokens), key **bridges**.
- **Exact + normalized match** of CTA/hook/opener across the set — the reliable catch (8× "Come to Him"; the "thousand years" hook family).
- **Whitelist rule (resolves the v2 contradiction):** the exact full-sentence CTA/hook/closer match **always runs and is never whitelisted**. The doctrinal whitelist ("the cross", "Christ", "grace") suppresses only **mid-body** shared-truth phrases in the *near-duplicate* layer — never a closer/opener. So A3 cannot be self-nullified.
- **Scope:** within-series AND across-catalogue, via a **catalogue manifest** (`data/catalogue_index.json`, auto-built by globbing narration folders) so discovery isn't manual.

**LLM backstop (the family/semantic layer):** `independent_review.py` gains `--type cluster` + `LENS_CLUSTER`. **Contract:** input = a manifest/list of folders; the CLI concatenates **spoken-only** text (via A1) of each into one prompt; verdict is **blocking** (a flagged cluster cannot lock); per-provider failure (e.g. grok) is tolerated as long as ≥3 reviewers return. Output under `_independent_review/<stamp>/` as today.

**Long-form:** same engine across its **movements** (intra-document) + against the catalogue.

### A3 — Acceptance test (gates the whole effort)
On the 8 real Psalm 22 shorts (in this repo), the deterministic A2 layer **must** flag: the **8× "Come to Him"** family (incl. "come to Him and drink" / "That's Jesus. Come to Him.") and the **"thousand years" hook family**. Committed as a fixture. **The exact-CTA catch is the hard requirement; the hook-family catch is verified against what's actually in the files (≈ #01/#02/#03/#08 strongly, #07 "a king once wrote… a thousand years before"), not an inflated 6/8 claim.** If A2 misses the CTA family, stop and fix A2 before building B/C.

## Phase B — KJV integrity (witnessed, scoped to used verses, fail-closed)

### B1 — Audit what's ACTUALLY corrupt now
`_audit_kjv_cache.py` dry-run lists current divergences (expected: Ps 22:7 comma; **not** Acts 8:35). Decisions follow the real list.

### B2 — A WITNESSED store of ONLY the verses in use (not the whole Bible)
**Problem (consensus).** A full-Bible witnessing with "human-adjudicate every disagreement" is unworkable (thousands of edition variances) and contradicts "prove cheap first."
**Design.**
- Build `data/kjv_verified.json` containing **only the refs actually quoted in shipped + planned narrations** (driven off A1's quote→ref map + B1).
- For each such verse: take **one named, vetted public-domain Cambridge-1769 structured dataset** as baseline (named + pinned in metadata), **spot-check it against a second independent source** and a trusted printed reference for the **known-hard cases** (Ps 22:7 comma, Matt 8:27 "!"); record the witness in metadata. No blanket full-Bible reconciliation.
- New verses entering a narration are witnessed on demand by the same step — bounded, incremental.

### B3 — True fail-closed (no fallback)
- Lock eligibility requires the quote's ref to be in `data/kjv_verified.json`. **bible-api is not a fallback for lock** — API text is marked **unverified** and **cannot satisfy a lock**. Unverifiable quote → **BLOCK** (not a soft WARN).
- `scripture.fetch_kjv` may still use the cache for *non-lock* engine context, but `cli_lock.py` only trusts the witnessed store.

### B4 — Ref-mapped, contiguous-substring check (interior elision + LXX-safe + ranges + normalization)
- Verify each quoted span as a **contiguous substring of its own tagged verse** (range refs joined across verse boundaries) in the witnessed store — catches **interior word omission**; **NT quotes checked against the NT verse**, never the Masoretic OT verse (no LXX false-fail).
- **Interior** punctuation must match; **boundary** ellipsis (`…`) allowed; **normalize** smart quotes/dashes/whitespace before compare (gemini). KJV italics handled by the chosen dataset's convention, documented.
- Threaded into `verify_narration.py` (the hand-authored path), not only `engine.review`.
